### 作业标题：多模态 Prompt 注入与简历自动化筛选

#### 1. 作业目标

- **附件处理**：练习将图片（模拟简历扫描件）作为多模态输入提交给 AI。
- **上下文注入**：实现基础的占位符替换（替代复杂的 AST），将招聘需求动态注入 Prompt。
- **JSON 结构化输出**：强制模型返回固定格式，便于程序自动解析打分。

├── filter_engine.py           # 【学生修改】核心逻辑：读取图片 -> 组装 Prompt -> 调用 API

    └── test_grading.py        # 自动打分脚本

#### 3. 核心任务设计

**任务一：配置动态筛选规则 (`config.json`)**
学生需要根据模拟的 HR 需求，设置筛选变量。

```json
{
  "target_role": "前端开发工程师",
  "must_have_skills": ["Vue 3", "Electron", "Vite"],
  "min_experience_years": 2
}
```

**任务二：实现 Prompt 注入逻辑 (`filter_engine.py`)**
学生需要编写一个简单的 Python 函数，将 `config.json` 的内容注入到预设的 Prompt 模板中。

- **要求**：使用 Python 的 `.format()` 或 `f-string` 完成替换。
- **多模态处理**：将 `resumes/` 下的图片转换为 Base64 编码，构造符合 Omni 模型规范的 Payload。

**任务三：结构化解析**
确保模型输出如下格式，否则自动打分系统将报错：

```json
{
  "match_score": 85,
  "reason": "具备 Vue 3 经验，但 Electron 实践较少",
  "decision": "Interview"
}
```

#### 5. 实验文档（README）简述

> ### 📝 实验：利用多模态 AI 构建简历初筛助手
>
> **场景**：你现在是博远信息技术社的 HR。我们收到了大量图片格式的简历。请编写脚本，利用多模态模型直接“看图取词”，并根据我们动态调整的招新标准进行打分。
>
> **你的挑战**：
>
> 1. **注入变量**：别让 AI 盲目筛选，把 `config.json` 里的要求塞进提示词里。
> 2. **处理多模态输入**：图片附件必须正确编码，否则 AI 会说“我看不见任何内容”。
> 3. **结果闭环**：确保 AI 只返回 JSON，因为后端的打分脚本要直接读取它。
